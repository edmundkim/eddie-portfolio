# Portfolio Reference Pack

Use this folder as a lightweight capture system for redesign references before anything turns into live UI.

## Working design preferences

- Current palette: [design-preferences.md](./design-preferences.md)

## How to use this pack

1. Add a new row to the log below whenever you find a strong reference.
2. Drop screenshots, GIFs, or recordings into `assets/` and note the filenames in the log.
3. If a reference is worth deeper analysis, copy `templates/reference-template.md` into `references/<slug>.md` and fill it in.
4. Borrow principles, pacing, and patterns. Do not copy exact visual treatment or branded details.

## Naming suggestions

- Note files: `references/<brand-or-site>-<page-or-pattern>.md`
- Asset files: `assets/<brand-or-site>-<what-it-shows>.<ext>`

## Status guide

- `captured`: saved, but not reviewed yet
- `studying`: worth unpacking in more detail
- `applied`: ideas from this reference influenced the portfolio
- `parked`: interesting, but not a current fit

## Quick capture checklist

When adding a new reference, try to capture at least:

- what makes it worth saving
- what part is actually relevant to your portfolio
- what should not be copied directly
- whether you need supporting screenshots or motion captures
